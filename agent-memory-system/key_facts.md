# Key Facts

> **TEMPLATE REQUIREMENT & SECURITY WARNING:**
> - **ONLY store non-sensitive information** (hostnames, ports, URLs, service names)
> - **NEVER store passwords, API keys, tokens, or credentials**
> - **Use environment variables and secret managers** for sensitive data
> - Organize by logical category (Database, APIs, Services, Local Setup, URLs)
> - Use bullet lists for clarity
>
> ⚠️ See [key_facts_template.md](references/key_facts_template.md) for what is safe to store.

## Local Development

- Ports, base URLs, and local service endpoints
- Required environment variables (names only; never values)
- Common paths and OS-specific notes

## Services

- Database hosts, ports, and names (non-sensitive)
- External service names and public endpoints

## URLs

- Public documentation links, dashboards, status pages
- Repository links (issues, pull requests)
